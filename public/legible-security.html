The economics

On-premise is now genuinely affordable.

Open-weight models have closed the quality gap with frontier models on SQL tasks — and the hardware to run them costs a fraction of what it did two years ago.

01
Model quality has caught up
Llama 3.1 70B, Mistral Large, and Qwen 2.5 Coder match GPT-4-class performance on NL-to-SQL benchmarks. SQL generation is a bounded, well-defined task — exactly where open models excel.
02
Hardware costs have collapsed
A single NVIDIA RTX 4090 (~£2,000) runs a 13B model comfortably. Cloud GPU spot instances for inference start at £0.20/hour. SQL queries take milliseconds — your inference costs are negligible.
03
Quantization makes models smaller
GGUF, AWQ, and GPTQ quantization cuts VRAM requirements by 50–75% with minimal quality loss. A model that once needed 40GB now runs in 8–12GB — standard mid-range GPU memory.
04
The tooling is production-ready
Ollama, vLLM, and llama.cpp are production-grade runtimes with enterprise support. If you can run a Docker container, you can run a local LLM. The complexity barrier is gone.

Regulatory & compliance

Built for regulated industries.

On-premise deployment directly satisfies the AI data processing requirements of the most demanding regulatory frameworks.

GDPR / UK GDPR
No international data transfer risk
Personal data processed by the LLM never leaves your jurisdiction. No Article 46 mechanism required. No adequacy decision dependency for AI processing.
Data residency satisfied
HIPAA
No BAA required with LLM provider
Protected Health Information implicit in query text never reaches a third-party Business Associate. On-premise removes the BAA requirement for the AI inference layer entirely.
PHI stays on-premise
SOC 2 Type II
AI inference within your trust boundary
All data processing occurs within your defined system boundary. LLM inference is treated identically to any other internal compute workload — no vendor risk assessment needed for the AI layer.
Trust boundary maintained
ISO 27001
Information transfer controls satisfied
On-premise deployment satisfies A.13 (information transfer), A.15 (supplier relationships), and A.10 (cryptography) without requiring vendor risk assessment for the LLM layer.
Controls A.13, A.15, A.10
FedRAMP / ITAR / DFARS
CUI and controlled technical data
For US federal contractors and defence suppliers, on-premise deployment in a FedRAMP-authorised environment satisfies CUI handling requirements. No commercial cloud LLM API currently meets these standards.
Air-gap compatible
FCA / PRA / MAS / FINRA
Third-party AI risk eliminated
Financial regulators are increasingly scrutinising third-party AI APIs for workloads involving customer data or MNPI. On-premise deployment removes this regulatory risk vector entirely before it becomes an enforcement issue.

Deployment options

Run it where you need it.

Legible integrates with any OpenAI-compatible inference endpoint — which means it works with every major local and private-cloud LLM runtime out of the box.

Ollama
Local · On-premise server
Zero-configuration local LLM serving. One command to download and run Llama 3.1, Mistral, Qwen 2.5, or Phi-3. Ideal for development, evaluation, and smaller teams.
Llama 3.1 Mistral Qwen 2.5 Phi-3
Recommended
vLLM
On-premise GPU server
Production-grade serving with high throughput, batching, and quantization support. The enterprise standard for self-hosted LLM inference. Full OpenAI API compatibility.
GPU optimised Batching Quantization
Private Cloud Endpoints
AWS · Azure · GCP (VPC isolated)
All major clouds offer managed private LLM endpoints with no data egress to model providers. Bedrock, Azure AI Studio, and Vertex AI all support VPC-isolated inference.
AWS Bedrock Azure AI Vertex AI
Air-Gapped Data Centre
Fully isolated network · No internet required
For defence, government, and regulated financial environments that require complete network isolation. Legible runs entirely offline — no external DNS lookups, no telemetry, no update pings. Delivered as a self-contained Docker image or Kubernetes chart.
